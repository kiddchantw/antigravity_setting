# 每日工作總結產生提示詞模板

## 基本提示詞

```
請為我產生 [日期] 的每日工作總結報告，針對 [資料夾路徑] 資料夾。

請執行以下步驟：

1. **分析 Git 提交記錄**
   - 查看 [日期] 當天的所有 commit
   - 分析每個 commit 的變更內容
   - 統計新增/修改/刪除的檔案和行數

2. **分析 Session 文件**
   - 找出 [日期] 當天建立或更新的 session 文件
   - 從 `docs/sessions/` 或 `[專案]/docs/sessions/` 目錄中查找
   - 提取每個 session 的目標、實作內容、技術決策

3. **統計程式碼變更**
   - 統計新增的檔案數量和行數
   - 統計修改的檔案數量和行數
   - 統計刪除的檔案數量
   - 統計測試檔案和測試數量

4. **整理功能開發內容**
   - 列出所有完成的功能模組
   - 每個功能包含：
     - Session 文件連結
     - 工作時間估算
     - 解決的問題
     - 實作內容（詳細步驟）
     - 技術亮點
     - 修改的檔案列表

5. **提取技術決策**
   - 從 session 文件中提取 Design Decisions
   - 記錄選擇的理由和權衡

6. **提取關鍵學習**
   - 從 session 文件的 Lessons Learned 中提取
   - 記錄解決方案和未來應用

7. **產生報告**
   使用以下格式產生 Markdown 報告：
   
   ```markdown
   # [專案名稱] - 今日工作總結報告
   
   **日期**: [日期] ([星期])
   **專案**: [專案名稱]
   **分支**: [分支名稱]
   **Git 提交**: [數量] 筆
   
   ## 📊 工作總覽
   [簡短描述今天的主要工作]
   
   ### 核心成就
   - ✅ [成就 1]
   - ✅ [成就 2]
   
   ## 🎯 功能開發詳情
   
   ### 1. [功能名稱] ⭐⭐⭐
   **Session**: `[session-file.md]`
   **工作時間**: ~[X] 小時
   **狀態**: ✅ 已完成
   
   #### 解決的問題
   [問題描述]
   
   #### 實作內容
   [詳細實作步驟]
   
   #### 技術亮點
   [技術重點]
   
   #### 修改檔案
   ```
   [檔案列表]
   ```
   
   ## 📁 檔案變更統計
   [統計表格]
   
   ## 🎯 技術決策
   [技術決策列表]
   
   ## 💡 關鍵學習
   [學習內容]
   
   ## ⏭️ 待辦事項
   [待辦清單]
   ```
```

## 進階提示詞（包含自動化指令）

```
請為我產生 [日期] 的每日工作總結報告，針對 [資料夾路徑] 資料夾。

**前置步驟**（請先執行）：
1. 執行 `cd [資料夾路徑] && git log --since="[日期] 00:00" --until="[日期] 23:59" --oneline` 查看當天提交
2. 執行 `cd [資料夾路徑] && git diff --stat [日期]^..[日期]` 統計變更
3. 查找 `docs/sessions/` 或 `[專案]/docs/sessions/` 目錄中日期為 [日期] 的檔案

**分析要求**：
1. **Git 提交分析**
   - 列出所有 commit hash 和訊息
   - 分析每個 commit 的檔案變更
   - 分類 commit 類型（feat/fix/docs/refactor 等）

2. **Session 文件分析**
   - 讀取所有相關 session 文件
   - 提取 Goal、實作內容、技術決策、Lessons Learned
   - 記錄每個功能的完成狀態

3. **程式碼統計**
   - 使用 `git diff --stat` 統計變更
   - 計算新增/修改/刪除的行數
   - 統計測試檔案和測試數量

4. **功能整理**
   - 按照重要性排序（⭐⭐⭐ 最重要）
   - 每個功能包含完整實作細節
   - 記錄技術亮點和創新點

5. **技術決策提取**
   - 從 session 文件的 "Design Decisions" 區塊提取
   - 記錄選擇的理由和 trade-offs

6. **學習內容提取**
   - 從 session 文件的 "Lessons Learned" 區塊提取
   - 記錄解決方案模式和未來應用

**報告格式**：
請參考 `工作總結-2025-12-05.md` 的格式和結構，產生相同風格的工作總結報告。
```

## 快速提示詞（簡化版）

```
請分析 [資料夾路徑] 資料夾在 [日期] 當天的工作內容，產生每日工作總結報告。

請：
1. 查看當天的 git commits
2. 找出當天的 session 文件
3. 統計程式碼變更
4. 整理功能開發內容
5. 提取技術決策和學習

報告格式請參考：工作總結-2025-12-05.md
```

## 使用範例

### 範例 1：Flutter 專案
```
請為我產生 2025-12-06 的每日工作總結報告，針對 a126_kompraa_flutter 資料夾。

請執行以下步驟：
1. 分析當天的 git commits
2. 找出 docs/sessions/2025-12/ 目錄中 06 開頭的 session 文件
3. 統計程式碼變更
4. 整理功能開發內容
5. 產生完整的工作總結報告

報告格式請參考：工作總結-2025-12-05.md
```

### 範例 2：Web 專案
```
請為我產生 2025-12-06 的每日工作總結報告，針對 A126-kompraa_web 資料夾。

請：
1. 查看當天的 git commits
2. 找出 docs_web/sessions/2025-12/ 目錄中的 session 文件
3. 統計程式碼變更
4. 整理功能開發內容
5. 產生工作總結報告
```

## 提示詞變數說明

- `[日期]`: 格式為 `YYYY-MM-DD`，例如 `2025-12-06`
- `[資料夾路徑]`: 相對於專案根目錄的路徑，例如 `a126_kompraa_flutter` 或 `A126-kompraa_web`
- `[專案名稱]`: 專案顯示名稱，例如 `A126 Kompraa (Flutter App)`
- `[分支名稱]`: 當前 git 分支，例如 `api_seller` 或 `main`

## 注意事項

1. **日期格式**: 使用 `YYYY-MM-DD` 格式
2. **Session 文件路徑**: 
   - Flutter: `a126_kompraa_flutter/docs/sessions/`
   - Web: `docs_web/sessions/`
3. **Git 指令**: 確保在正確的資料夾內執行 git 指令
4. **時區**: 注意 git commit 的時區，可能需要調整日期範圍
